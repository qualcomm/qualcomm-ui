### Clear Trigger

- When using the simple API, the clear button renders automatically. Change this by setting the [clearable](./#clearable) prop to false.
- When using the composite API, you render the `q-text-input-clear-trigger` declaratively within the input group.
- The clear button only shows when the input field has text.

<QdsDemo name="TextInputClearTriggerDemo" />

### Error Text and Indicator

Error messages are displayed using two props:

- [invalid](./#invalid)
- [errorText](./#errorText) (or the `q-text-input-error-text` directive when using the composite API)

The error text and indicator will only render when [invalid](./#invalid) is true.

<QdsDemo name="TextInputErrorTextDemo" />

## Forms

Control and validate the input value using Angular form control bindings.

### Template Forms

When using template-driven forms with `ngModel`, perform validation manually in your component and pass the result to the [invalid](./#invalid) property.

> [!tip]
> Form validation timing is controlled by the [updateOn](https://angular.dev/api/forms/FormControl#updateOn) property on the form control.

<QdsDemo name="TextInputTemplateFormsInvalidDemo" />

#### States

When using template forms, the [disabled](./#disabled), [readOnly](./#readOnly), and [invalid](./#invalid) properties govern the interactive state of the control.

<QdsDemo name="TextInputStatesDemo" />

#### Required

When using template forms, pass the [required](./#required) property to apply the [RequiredValidator](https://angular.dev/api/forms/RequiredValidator) to the form control.

<QdsDemo name="TextInputRequiredTemplateFormsDemo" />

### Reactive Forms

Use Reactive Forms for better control of form state and validation.

<QdsDemo name="TextInputReactiveFormsDemo" />

#### State Guidelines

The [disabled](./#disabled), [invalid](./#invalid), and [required](./#required) properties have no effect when using Reactive Forms. Use the equivalent Reactive Form bindings instead:

<QdsDemo name="TextInputReactiveFormStatesDemo" />

### Composite API & Forms

When using the composite API with Angular Forms, always apply form control bindings to the `q-text-input-root` directive.

<QdsDemo name="TextInputCompositeFormsDemo" />
