# {frontmatter.title}

The accordion component enables users to collapse and expand multiple content sections within a limited space. This helps group and hide content effectively, maintaining a clean interface and reducing clutter.

```typescript
import {AccordionModule} from "@qualcomm-ui/angular/accordion"
```

## Usage

- The accordion requires each of its items to have a unique `value` input.
- The accordion is keyboard-navigable once focused. Arrow keys can be used to move to the next or previous item. Home and End keys can be used to jump to the first or last item, respectively.

### Multiple

By default, only a single accordion section can be open at a time. Setting [multiple](./#multiple-1) allows multiple sections to be open or closed simultaneously.

### Collapsible

When [multiple](./#multiple-1) is not set, enabling [collapsible](./#collapsible-1) lets the user close an open accordion item by clicking it again.

<QdsDemo name="AccordionCollapsibleDemo" />

### Default Value

The [defaultValue](./#defaultValue) input allows you to set the initial state for the accordion by specifying which accordion item or items (if [multiple](./#multiple-1) is set) should be open by default.
It expects an array of values that should match those passed to each accordion item.

<QdsDemo name="AccordionDefaultValueDemo" />

### Disabled

You can disable accordion items by setting their [disabled](./#disabled-2) input. This prevents the item from being interacted with.
You can disable the entire accordion by setting the [disabled](./#disabled-1) input on the accordion root element.

<QdsDemo name="AccordionDisabledDemo" />

### Uncontained

This input removes the default horizontal padding from accordion items, allowing them to sit seamlessly within the surrounding content.

<QdsDemo name="AccordionUncontainedDemo" />

### Size

Use the [size](./#size-1) input to control the size of the accordion items. The available sizes are `sm`, `md` (default), and `lg`.

<QdsDemo name="AccordionSizeDemo" />

### Secondary Text

Use the `q-accordion-item-trigger-secondary` directive to display additional text opposite the trigger title.

<QdsDemo name="AccordionSecondaryTextDemo" />

### Icon

Pass an icon to the [icon](./#icon-1) input on the `q-accordion-item-trigger` element to display it next to the trigger title.

<QdsDemo name="AccordionIconDemo" />

## Shortcuts

### Item Content

The `q-accordion-item-content` directive is a shortcut for the following:

```angular-html
<div q-accordion-item-content-animator>
  <div q-accordion-item-content-body>
    <ng-content />
  </div>
</div>
```
