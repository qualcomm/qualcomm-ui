## Overview

The drawer's API is similar to the [Dialog](/components/dialog) component.

## Examples

### Placement End

The drawer's default [placement](./#placement) is `end`, which maps to the right side of the screen in LTR languages and the left side in RTL languages.

<Demo component={demos.DrawerPlacementDemo} />

### Placement Start

Use [placement](./#placement) `start` to change the placement to the left side of the screen.

<Demo component={demos.DrawerPlacementStartDemo} />

### Controlled State

Drawer visibility can be controlled using the [open](./#open), [onOpenChange](./#onOpenChange) and [defaultOpen](./#defaultOpen) props. These props follow our [controlled state](/patterns/controlled-state) pattern.

<Demo component={demos.DrawerControlledStateDemo} />

### Custom Container

Use the `container` prop of the `Portal` component to render the drawer in a custom container. You'll need to override the positioner's default styles to position the drawer correctly.

Consider setting [closeOnInteractOutside](./#closeOnInteractOutside) to `false` to prevent the drawer from closing when interacting outside the drawer.

<Demo component={demos.DrawerCustomContainerDemo} expanded />

## Shortcuts

### \<Drawer.FloatingPortal\>

A helper component that combines the portal, positioner, and content
components. This shortcut is equivalent to:

```tsx
<Portal {...props}>
  <Drawer.Backdrop {...backdropProps} />
  <Drawer.Positioner {...positionerProps}>
    <Drawer.Content {...contentProps}>{children}</Drawer.Content>
  </Drawer.Positioner>
</Portal>
```
