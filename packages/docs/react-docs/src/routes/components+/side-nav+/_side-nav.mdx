## Overview

The side nav is an extension of our [tree](/components/tree) component.

- The Side Navigation relies on the `TreeCollection` class to manage its items. Refer to the [API](./#tree-collection) below for details.
- Like with the tree, Side Navigations are composed of nodes, which are objects that describe the navigation SideNav. There are two types of nodes:
- A `branch` node has children.
- A `leaf` node does not have any children.
- Each node has a `value` (unique identifier used for expansion) and a `text` (display text).

**Default object shape:**

- `value` (required): unique identifier for expansion/selection
- `text` (required): display text
- `nodes` (optional): child nodes for branches
- `disabled` (optional): prevents interaction when `true`

These keys can be customized via the `TreeCollection` [constructor](./#constructor).

## Examples

### Node Shorthand

We expose the `<SideNav.Nodes>` shorthand for rendering nodes. Use its [renderBranch](./#renderBranch) and [renderLeaf](./#renderLeaf) render props to customize the content of each tree node.

Note that `<SideNav.Nodes>` automatically renders child nodes for branches, so you only have to customize the content of the node itself.

<QdsDemo name="SideNavNodeShorthandDemo" />

### Grouping

Group nodes using the collection's [groupChildren](./#groupChildren) method. This should be called on the top-level node of your SideNav. Nested groups are not supported.

<QdsDemo name="SideNavGroupsDemo" />

### Default Expanded

Expand nodes by default using the [defaultExpandedValue](./#defaultExpandedValue) prop. Or use [expandedValue](./#expandedValue) and [onExpandedChange](./#onExpandedChange) to control the expansion manually. These props follow our [controlled state](/patterns/controlled-state) pattern.

<QdsDemo name="SideNavDefaultExpandedDemo" />

### Disabled Nodes

You can disable nodes by setting the `disabled` property on the node object.

<QdsDemo name="SideNavDisabledNodeDemo" />

### Filtering

Here's an example that filters the items using `matchSorter`.

<QdsDemo name="SideNavFilteringDemo" />

### Links

Side Nav nodes can be links using [polymorphic composition](/polymorphic-components). Use the `render` prop on the `<SideNav.LeafNode>` to specify the element type.

<QdsDemo name="SideNavLinksDemo" />

### Collapsed

Side Nav can be collapsed to render only the top-level icons.

The open state of the panel can be controlled using the [open](./#open), [onOpenChange](./#onOpenChange) and [defaultOpen](./#defaultOpen) props. These props follow our [controlled state](/patterns/controlled-state) pattern.
