# {frontmatter.title}

Badges provide immediate visual context through color-coded indicators that help users quickly identify status, priority levels, or content categories. They're designed to be concise, accessible, and consistent across desktop and mobile experiences. Use badges to draw attention to important information without overwhelming the interface.

```typescript
import {Badge} from "@qualcomm-ui/react/badge"
```

Badge supports four distinct types via the `type` prop: `text`, `count`, `icon`, and `status`.

## Text Badges

Text badges display text labels and are the default badge type.

<QdsDemo name="BadgeTextDemo" />

### Emphasis

Text badges support semantic and category emphasis via the [emphasis](./#emphasis-3) prop. Semantic emphasis options include `neutral`, `brand`, `info`, `success`, `warning`, and `danger`. Category emphasis options range from `cat-1` through `cat-10`.

<QdsDemo name="BadgeTextEmphasisDemo" />

### Variant

Control the visual weight with the [variant](./#variant-3) prop. Text badges support `default` (bolder appearance) and `subtle` (lighter appearance) variants.

<QdsDemo name="BadgeTextVariantsDemo" />

### Sizes

Change the text badge dimensions using the [size](./#size) prop. Three sizes are available: `sm`, `md` (default), and `lg`.

<QdsDemo name="BadgeTextSizesDemo" />

## Count Badges

Count badges display numeric values. Use the [count](./#count) prop for auto formatting, or use [children](./#children-1) for manual content.

<QdsDemo name="BadgeCountDemo" />

### Auto Formatting

When using the [count](./#count) prop with [max](./#max), the badge automatically formats overflow values. If count exceeds max, the badge displays "max+" and applies special styling. The default max is 99.

<QdsDemo name="BadgeCountMaxDemo" />

### Variants

Count badges support multiple variants via the [variant](./#variant-4) prop: `neutral` (default), `neutral-outline`, `brand`, `brand-outline`, `info`, `success`, `warning`, `danger`, `persistent-black` and `persistent-white`.

<QdsDemo name="BadgeCountVariantsDemo" />

### Sizes

Change the count badge dimensions using the [size](./#size-1) prop. Three sizes are available: `sm`, `md` (default), and `lg`.

<QdsDemo name="BadgeCountSizesDemo" />

## Icon Badges

Icon badges display icons. Use the [icon](./#icon) prop to provide a Lucide icon, or use [children](./#children-2) for custom SVG content.

<QdsDemo name="BadgeIconDemo" />

### Emphasis

Icon badges support the same semantic and category emphasis options as text badges via the [emphasis](./#emphasis-4) prop.

<QdsDemo name="BadgeIconEmphasisDemo" />

### Variant

Icon badges support `default` and `subtle` variants via the [variant](./#variant-5) prop.

<QdsDemo name="BadgeIconVariantsDemo" />

### Sizes

Change the icon badge dimensions using the [size](./#size-2) prop. Six sizes are available: `xxs`, `xs`, `sm`, `md` (default), `lg`, and `xl`.

<QdsDemo name="BadgeIconSizesDemo" />

## Status Badges

Status badges display a colored dot indicator with no content. They're useful for showing online/offline status or availability states.

### Emphasis

Status badges support semantic emphasis via the [emphasis](./#emphasis-5) prop: `neutral`, `brand`, `info`, `success`, `warning`, and `danger`.

<QdsDemo name="BadgeStatusEmphasisDemo" />

### Variant

Control the style with the [variant](./#variant-6) prop. Status badges support `filled` (default) and `outlined` variants.

<QdsDemo name="BadgeStatusVariantsDemo" />

### Sizes

Change the status badge dimensions using the [size](./#size-3) prop. Five sizes are available: `xs`, `sm`, `md` (default), `lg`, and `xl`.

<QdsDemo name="BadgeStatusSizesDemo" />
