### States

Based on the inputs, the checkbox will render as checked, indeterminate, or unchecked (default). The checked state takes precedence over indeterminate.

::: spoiler State Table

| checked | indeterminate | Result State                                                     |
| ------- | ------------- | ---------------------------------------------------------------- |
| `false` | `false`       | <Checkbox label="unchecked" checked={false} />                   |
| `false` | `true`        | <Checkbox label="indeterminate" indeterminate checked={false} /> |
| `true`  | `false`       | <Checkbox label="checked" checked />                             |
| `true`  | `true`        | <Checkbox label="checked" indeterminate checked />               |

:::

<Demo name="CheckboxStatesDemo" component={demos.CheckboxStatesDemo} />

## Composite Guidelines

The composite elements are only intended to be used as direct descendants of the `<Checkbox.Root>` component.

```tsx
/* Won't work alone ❌ */
<Checkbox.HiddenInput />
<Checkbox.Control />

/* Works as expected ✅ */
<Checkbox.Root>
  <Checkbox.HiddenInput />
  <Checkbox.Control />
</Checkbox.Root>
```
