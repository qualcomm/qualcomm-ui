# {frontmatter.title}

A styled and accessible password input component that captures sensitive user credentials with masked text display.

```tsx
import {PasswordInput} from "@qualcomm-ui/react/password-input"
```

## Examples

### Simple

The simple API bundles all subcomponents together into a single component.

<Demo component={demos.PasswordInputSimpleDemo} />

### Icons (Simple)

Add a [startIcon](./#startIcon) to the simple `<PasswordInput>` using the prop at the root. Unlike the [`<TextInput>`](/components/text-input), the `endIcon` is not supported because its slot is reserved for the password visibility trigger.

<Demo component={demos.PasswordInputIconsDemo} />

### Composite

The composite pattern provides full control over the component's elements by composing individual subcomponents.

<Demo component={demos.PasswordInputCompositeDemo} />

### Controlled Value

Set the initial value using the [defaultValue](./#defaultValue) prop, or use [value](./#value) and [onValueChange](./#onValueChange) to control the value manually. These props follow our [controlled state](/patterns/controlled-state) pattern.

<Demo expanded component={demos.PasswordInputControlledValueDemo} />

### Controlled Visibility

Set the initial visibility using the [defaultVisible](./#defaultVisible) prop, or use [visible](./#visible) and [onVisibleChange](./#onVisibleChange) to control the visibility manually. These props follow our [controlled state](/patterns/controlled-state) pattern.

<Demo expanded component={demos.PasswordInputControlledVisibilityDemo} />
