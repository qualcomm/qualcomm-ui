# {frontmatter.title}

This library requires React 19+.

There are four quick steps to installation:

1. Install the npm packages
2. Import the CSS
3. Configure fonts
4. Configure theme provider

This guide assumes you're using Next.js with the app router. For page router support, see the corresponding [next-themes](https://www.npmjs.com/package/next-themes) documentation in step 3.

## 2. Import the CSS

In your `app/globals.css` file, import the QUI CSS.

```css
@import "@qualcomm-ui/qds-core/styles/components.css";
@import "@qualcomm-ui/qds-core/themes/qualcomm-dark.css";
@import "@qualcomm-ui/qds-core/themes/qualcomm-light.css";
```

If you're using Tailwind, import the theme CSS at the `components` layer instead:

```css
@import "tailwindcss";

/* Important: always order these imports after the tailwindcss import */
@import "@qualcomm-ui/qds-core/styles/components.css" layer(components);
@import "@qualcomm-ui/qds-core/themes/qualcomm-light.css" layer(components);
@import "@qualcomm-ui/qds-core/themes/qualcomm-dark.css" layer(components);
```

Learn more about the available brands in the [theme documentation](/theming/overview).

## 3. Configure fonts and theme provider

QUI components are designed with the Roboto Flex and Roboto Mono font families.

In Next.js, configure fonts using `next/font/google` and set up the theme provider in your root layout:

```tsx
import type {Metadata} from "next"
import {Roboto_Flex, Roboto_Mono} from "next/font/google"
import "./globals.css"
import {type ReactNode} from "react"

const sansFont = Roboto_Flex({
  variable: "--font-roboto-flex",
  weight: ["400", "500", "600"],
  subsets: ["latin"],
})

const monoFont = Roboto_Mono({
  variable: "--font-roboto-mono",
  subsets: ["latin"],
})

export const metadata: Metadata = {
  title: "Your App Title",
  description: "Your app description",
}

export default function RootLayout({
  children,
}: Readonly<{
  children: ReactNode
}>) {
  return (
    // preview
    <html lang="en">
      <body className={`${sansFont.variable} ${monoFont.variable}`}>
        {children}
      </body>
    </html>
    // preview
  )
}
```

## 4. Configure the theme provider

QUI styles support both light and dark modes. Follow this step to configure storage for theme switching so that the user's preferred theme is remembered when they return to the site.

From the previous step, add the `ThemeProvider` to your root layout:

{/* eslint-disable-next-line prettier/prettier */}
```tsx
// previous imports and setup...
import {ThemeProvider} from "next-themes"; // [!code ++]

<html lang="en"> // [!code --]
<html lang="en" data-brand="qualcomm" suppressHydrationWarning> // [!code ++]
  <body className={`${sansFont.variable} ${monoFont.variable}`}>
    <ThemeProvider attribute="data-theme">{children}</ThemeProvider>
  </body>
</html>
```

Key configuration details:

- `suppressHydrationWarning` on `<html>` prevents hydration warnings from theme switching
- `data-brand="qualcomm"` sets the brand attribute (change to `"snapdragon"` or `"dragonwing"` to use those brands instead)
- `ThemeProvider` with `attribute="data-theme"` enables theme switching via data attributes

### (Optional) Customize the default theme

The default theme is determined by user system preferences.

Note that setting a different default theme is [currently bugged](https://github.com/pacocoursey/next-themes/issues/240). When that bug is fixed, you can set a different default theme using the ThemeProvider:

```tsx
<ThemeProvider attribute="data-theme" defaultTheme="dark" enableSystem={false}>
  {children}
</ThemeProvider>
```

To set a different brand, change the `data-brand` attribute on the `<html>` element:

```tsx
<html lang="en" suppressHydrationWarning data-brand="snapdragon">
```
